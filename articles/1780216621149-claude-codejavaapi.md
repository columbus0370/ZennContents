## はじめに
- 背景：レガシーJavaプロジェクトでAPI仕様書が古く、ドキュメント整備に工数がかかっている課題
- 目的：Claude Codeを活用してソースコードからOpenAPI定義を自動生成できるか検証

## 検証環境
- Java 11 / Spring Boot 2.x の既存プロジェクト
- Claude Code（Anthropic）
- 対象：約50エンドポイント

## 実験手順
1. Controller層のソースコードをClaude Codeに読み込ませる
2. 「このJavaコードからOpenAPI 3.0形式のYAMLを生成して」とプロンプト
3. 出力されたYAMLをSwagger Editorで検証
4. 既存ドキュメントとの差分を確認

## 結果と課題
- 基本的なエンドポイント定義は約80%の精度で生成可能
- カスタムアノテーションや業務固有のバリデーションは手動補正が必要
- 一括処理ではなく段階的に確認しながら進めるのが現実的

## 学び
- 完全自動化は難しいが「たたき台生成」としては十分実用的
- プロンプト設計次第で精度が大きく変わる

## まとめ
- 2日→2時間への工数削減の可能性を確認
- 今後は他プロジェクトでも検証予定
